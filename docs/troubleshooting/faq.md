# 자주 묻는 질문 (FAQ)

## 일반 질문

### Q: PromptBlocks는 어떤 AI 이미지 생성 도구와 함께 사용하나요?

**A:** Midjourney, DALL-E 3, Stable Diffusion 등 텍스트 기반 이미지 생성 AI 모두와 함께 사용할 수 있습니다. PromptBlocks는 프롬프트를 체계적으로 관리하는 도구로, 어떤 AI 도구를 사용하든 완성된 프롬프트를 복사해서 사용하기만 하면 됩니다.

### Q: 무료로 사용할 수 있나요?

**A:** 네, 무료 플랜으로 시작할 수 있습니다. 무료 플랜에서는:

- 월 20회 이미지/텍스트 분해
- 최대 50개 블록 저장
- 기본 기능 모두 사용 가능

더 많은 사용량이 필요하시면 Pro 플랜으로 업그레이드하세요.

### Q: API 키는 어디서 설정하나요?

**A:** 좌측 메뉴의 **설정** 페이지에서 OpenAI(GPT) 또는 Google Gemini API 키를 입력할 수 있습니다. API 키는 브라우저의 로컬 스토리지에 안전하게 저장되며, 서버에는 저장되지 않습니다.

### Q: 비로그인 상태에서도 사용할 수 있나요?

**A:** 네, 비로그인 상태에서도 블록과 프리셋을 localStorage에 저장하여 사용할 수 있습니다. 다만 브라우저 데이터를 삭제하면 저장된 내용이 사라지므로, 중요한 데이터는 로그인하여 클라우드에 저장하시는 것을 권장합니다.

### Q: 데이터는 안전하게 보관되나요?

**A:** 네:

- 모든 데이터는 암호화된 연결(HTTPS)으로 전송됩니다
- Supabase 데이터베이스에 안전하게 저장됩니다
- API 키는 브라우저 로컬 스토리지에만 저장되며 서버로 전송되지 않습니다
- 결제 정보는 Stripe에서 직접 관리됩니다

## Decompose (분해) 관련

### Q: 분석 결과가 부정확해요. 어떻게 개선할 수 있나요?

**A:** 다음 방법을 시도해보세요:

1. **더 나은 이미지 사용**: 선명하고 복잡하지 않은 이미지 선택
2. **수동 편집**: 분핸된 블록 내용을 직접 수정
3. **다른 AI 모델 시도**: GPT와 Gemini 중 다른 모델로 테스트
4. **텍스트 모드 사용**: 이미지 대신 텍스트로 직접 입력

### Q: 배치 분석은 어떻게 하나요?

**A:** Decompose 페이지에서 여러 이미지를 동시에 업로드할 수 있습니다 (최대 5개). 각 이미지는 순차적으로 분석되며, 진행 상태를 실시간으로 확인할 수 있습니다.

### Q: 분석에 사용되는 AI 모델은 무엇인가요?

**A:** 현재 지원되는 모델:

- **OpenAI**: GPT-5.2, GPT-5-mini, GPT-4o
- **Google**: Gemini 3 Pro, Gemini 3 Flash, Gemini 2.5 Flash, Gemini 2.0 Flash

모델별로 분석 스타일이 다를 수 있으니 여러 모델을 테스트해보세요.

## Library (라이브러리) 관련

### Q: 블록에 태그는 어떻게 추가하나요?

**A:** 현재 태그는 자동으로 추출됩니다. 블록 저장 시 AI가 내용을 분석하여 관련 태그를 자동으로 할당합니다. 태그 클라우드를 통해 태그별로 블록을 필터링할 수 있습니다.

### Q: 컬렉션 간 블록 이동은 어떻게 하나요?

**A:**

1. Library 페이지에서 이동할 블록 선택
2. "이동" 버튼 클릭
3. 대상 컬렉션 선택

또는 블록을 드래그하여 원하는 컬렉션에 드롭할 수도 있습니다.

### Q: 블록을 삭제하면 복구할 수 있나요?

**A:** 현재 삭제된 블록은 즉시 완전히 삭제됩니다. 중요한 블록은 삭제 전에 백업을 권장합니다. 설정 페이지에서 데이터를 JSON으로 내보내기할 수 있습니다.

## Assemble (조립) 관련

### Q: 무작위(randomize) 기능은 어떻게 작동하나요?

**A:** 무작위 버튼을 클릭하면:

1. **잠금되지 않은 블록만** 대상이 됩니다
2. 선택된 컬렉션 범위 내에서만 후보가 선택됩니다
3. 같은 블록 타입 내에서만 교체됩니다 (예: 스타일 블록은 다른 스타일 블록으로)

처음부터 조립할 때는 "전체 블록" 범위로 시작하세요.

### Q: 프리셋과 스냅샷의 차이는 무엇인가요?

**A:**

- **프리셋**: 재사용 가능한 블록 조합 템플릿. 이름을 붙여 저장하고 나중에 불러올 수 있습니다.
- **스냅샷**: 특정 시점의 조립 상태를 저장. 히스토리처럼 사용하며, 과거 상태로 되돌아갈 수 있습니다.

### Q: 얼굴 일관성(Face Consistency)은 어떻게 설정하나요?

**A:**

1. Decompose에서 캐릭터 이미지 분석
2. "얼굴 기준 저장" 클릭
3. Assemble에서 해당 블록들은 자동으로 잠금됨
4. 다른 블록들만 무작위로 변경하며 일관된 캐릭터 유지

### Q: Midjourney 파라미터(--sref, --sw 등)는 어떻게 추가하나요?

**A:**

1. Assemble 페이지에서 "Output Mode"를 "Midjourney"로 변경
2. SREF 코드, Style Weight, Style Version 입력
3. 프롬프트가 자동으로 파라미터와 함께 생성됨

## 기술적 질문

### Q: 어떤 브라우저를 지원하나요?

**A:** 최신 버전의 Chrome, Firefox, Safari, Edge를 지원합니다. Internet Explorer는 지원하지 않습니다.

### Q: 모바일에서도 사용할 수 있나요?

**A:** 네, 반응형 웹 디자인으로 모바일 브라우저에서도 사용할 수 있습니다. 다만 이미지 업로드와 상세한 편집은 데스크탑에서 더 편리합니다.

### Q: 데이터를 내보내기/가져오기 할 수 있나요?

**A:** 네, 설정 페이지에서:

- **내보내기**: 모든 블록과 프리셋을 JSON 파일로 다운로드
- **가져오기**: JSON 파일을 업로드하여 데이터 복원

## 문제 해결

### Q: "API 키가 설정되지 않았습니다" 오류가 떠요.

**A:** 설정 페이지에서 유효한 OpenAI 또는 Gemini API 키를 입력했는지 확인하세요. API 키는 해당 AI 서비스의 웹사이트에서 발급받을 수 있습니다.

### Q: 이미지 업로드가 안돼요.

**A:** 다음을 확인하세요:

- 파일 크기가 20MB 이하인지
- 이미지 형식(JPG, PNG, WebP)인지
- 네트워크 연결 상태

### Q: 페이지가 느리게 로딩돼요.

**A:**

- 브라우저 캐시 삭제
- 불필요한 블록 정리 (500개 이상 저장 시 성능 저하 가능)
- Chrome의 경우 하드웨어 가속 활성화 확인

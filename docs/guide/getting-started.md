# PromptBlocks 시작하기

PromptBlocks는 AI 이미지 생성 프롬프트를 체계적으로 관리하고 최적화하는 워크스페이스입니다. 이 문서에서는 PromptBlocks의 핵심 개념과 효과적인 사용법을 설명합니다.

---

## 🎯 PromptBlocks란?

PromptBlocks는 다음과 같은 문제를 해결합니다:

- **긴 프롬프트 관리**: 복잡한 프롬프트를 의미 단위로 분해
- **재사용성**: 잘 작동하는 프롬프트 구성요소를 라이브러리화
- **일관성**: 캐릭터나 스타일의 일관된 결과물 생성
- **협업**: 팀 단위 프롬프트 자산 공유

---

## 🧩 핵심 개념: 3단계 워크플로우

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  DECOMPOSE  │ ──► │   LIBRARY   │ ──► │   ASSEMBLE  │
│  (분해)     │     │  (라이브러리) │     │   (조립)    │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 1️⃣ Decompose (분해)

이미지나 텍스트를 13개의 구조적 블록으로 분해합니다.

**13개 블록 타입**:
| 블록 | 설명 | 예시 |
|------|------|------|
| 주제 유형 | 피사체 종류 | portrait, landscape |
| 스타일 | 시각적 스타일 | photorealistic, anime |
| 인물 외형 | 외모 특징 | long hair, blue eyes |
| 의상 | 옷과 액세서리 | white dress, sneakers |
| 포즈/표정 | 자세와 감정 | sitting, smiling |
| 소품/오브젝트 | 함께 등장하는 물건 | book, coffee cup |
| 배경/환경 | 장소와 배경 | city street, forest |
| 조명 | 빛의 종류와 방향 | golden hour, soft light |
| 칩처/렌즈 | 촬영 기법 | close-up, 85mm lens |
| 색감/분위기 | 전체적인 톤 | warm tones, cinematic |
| 이미지 내 텍스트 | 포함된 글자 | "Hello", neon sign |
| 구도 | 화면 배치 | rule of thirds, centered |
| 기술 파라미터 | 해상도, 비율 | 8k, --ar 16:9 |

> 💡 **팁**: 모든 이미지가 13개 블록을 모두 가지는 것은 아닙니다. 이미지에 따라 일부 블록은 비어있을 수 있습니다.

### 2️⃣ Library (라이브러리)

분핸된 블록을 저장하고 관리하는 공간입니다.

**주요 기능**:

- **컬렉션**: 블록을 주제별로 그룹화
- **태그**: 블록에 키워드 추가하여 검색 용이
- **즐겨찾기**: 자주 사용하는 블록 표시
- **검색**: 내용이나 태그로 빠르게 찾기

**사용 예시**:

```
📁 내 컬렉션
├── 📁 인물 초상화
│   ├── ✨ "beautiful Korean woman" (즐겨찾기)
│   ├── "soft lighting"
│   └── "85mm lens"
├── 📁 풍경
│   ├── "golden hour"
│   └── "mountain backdrop"
└── 📁 미분류
    └── "cinematic color grading"
```

### 3️⃣ Assemble (조립)

라이브러리의 블록을 조합하여 완성된 프롬프트를 만듭니다.

**주요 기능**:

- **드래그앤드롭**: 블록 순서 변경
- **잠금**: 고정할 블록 보호 (무작위 교체 시 유지)
- **무작위 조합**: 선택된 범위 내에서 블록 랜덤 교체
- **작가 스타일**: 50+ 아티스트 스타일 적용
- **Midjourney 파라미터**: --sref, --sw 등 고급 설정

## 🔄 실제 사용 시나리오

### 시나리오 1: 이미지 분석 및 재생산

> "Pinterest에서 본 이미지를 비슷하게 만들고 싶어요"

**단계**:

1. **Decompose**: 이미지 업로드 → AI가 13개 요소 분석
2. **Library**: 분핸된 블록 중 유용한 것 저장
3. **Assemble**: 저장한 블록 조합 + 스타일 추가
4. **Copy**: 완성된 프롬프트를 Midjourney에 붙여넣기

### 시나리오 2: 캐릭터 일관성 유지

> "같은 캐릭터를 다른 상황에서 여러 장 만들고 싶어요"

**단계**:

1. **Reference 설정**: 기준이 될 캐릭터 이미지 분석
2. **얼굴 앵커 저장**: "한국 여성, 긴 검은 머리, 갈색 눈" 등 저장
3. **블록 잠금**: Subject Type, Appearance 블록 고정
4. **무작위 조합**: 배경, 의상, 포즈만 변경하여 다양한 상황 생성

### 시나리오 3: 팀 단위 프롬프트 관리

> "우리 팀이 자주 쓰는 스타일을 표준화하고 싶어요"

**단계**:

1. **컬렉션 생성**: "브랜드 스타일 가이드" 등의 컬렉션 만들기
2. **공유**: 팀원들과 컬렉션 공유
3. **프리셋 저장**: 자주 쓰는 블록 조합을 프리셋으로 저장
4. **공유 링크**: 완성된 프롬프트를 읽기 전용 링크로 공유

## 💡 모범사례

### 1. 블록 저장 기준

**저장하면 좋은 것**:

- ✅ 재사용 가능한 구성요소
- ✅ 프로젝트별 핵심 스타일
- ✅ 잘 작동하는 조합

**저장하지 않아도 되는 것**:

- ❌ 한 번만 쓰이는 특정 상황
- ❌ 테스트용 블록
- ❌ 품질이 낮은 추출 결과

### 2. 효과적인 컬렉션 구성

**추천 구조**:

```
📁 프로젝트별
├── 📁 클라이언트 A 캠페인
├── 📁 남성복 브랜드
└── 📁 시즌 2024 Spring

📁 주제별
├── 📁 인물 초상화
├── 📁 제품 사진
├── 📁 풍경
└── 📁 추상적 이미지

📁 스타일별
├── 📁 사실적 (Photorealistic)
├── 📁 애니메이션
└── 📁 빈티지/레트로
```

### 3. 프롬프트 조립 팁

**순서 중요성**:

1. 주제 유형 (누구/무엇인가)
2. 외형/의상 (어떤 모습인가)
3. 포즈/표정 (무엇을 하는가)
4. 배경/환경 (어디에서)
5. 조명/카메라 (어떻게 촬영)
6. 스타일/색감 (어떤 느낌)

**Midjourney 팁**:

- 주제는 앞에, 스타일은 뒤에
- 중요한 키워드는 ::weight로 강조
- --sref로 일관된 스타일 유지

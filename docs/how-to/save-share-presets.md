# 프리셋 저장하고 공유하기

자주 쓰는 블록 조합을 프리셋으로 저장하고 팀원들과 공유하는 방법을 알아봅니다.

## 개요

**프리셋(Preset)**은 자주 사용하는 블록 조합을 이름 붙여 저장하는 기능입니다. 같은 스타일이나 구조를 반복해서 사용할 때 시간을 절약할 수 있습니다.

## 단계별 가이드

### 1. 프리셋 저장

**Assemble에서 저장**:

1. 블록 조합 완성
2. **"프리셋 저장"** 버튼 클릭
3. 프리셋 이름 입력 (예: "브랜드 A 스타일")
4. 저장 완료

**저장되는 내용**:

- 블록 조합 및 순서
- 선택된 스타일 아티스트
- Midjourney 파라미터 설정

### 2. 프리셋 불러오기

1. **"프리셋"** 버튼 클릭 (Assemble 페이지 상단)
2. 저장된 프리셋 목록 확인
3. 원하는 프리셋 클릭
4. 블록들이 자동으로 로드됨

### 3. 프리셋 관리

**수정**:

1. 프리셋 불러오기
2. 블록 조합 수정
3. 새 이름으로 저장 (덮어쓰기 불가)

**삭제**:

1. 프리셋 목록에서 삭제 아이콘 클릭
2. 확인 후 삭제

### 4. 공유 링크 생성

**읽기 전용 공유**:

1. Assemble에서 **"공유"** 버튼 클릭
2. 공유 링크 자동 생성
3. 링크 복사
4. 팀원에게 공유

**링크 포함 내용**:

- 완성된 프롬프트
- 사용된 블록 목록
- 네거티브 프롬프트
- 블록 수

### 5. 링크로 프롬프트 보기

공유받은 링크로 접속하면:

- 프롬프트 내용 확인 가능
- 복사 버튼으로 클립보드에 복사
- 블록 구성 확인
- (로그인 시) 블록을 내 라이브러리에 저장 가능

## 팁과 모범사례

### ✅ 효과적인 프리셋 명명

- **목적 중심**: "Portrait Style"보다 "Corporate Headshot Style"
- **버전 관리**: "Spring Campaign v1", "Spring Campaign v2"
- **팀 공유용**: 팀명 또는 프로젝트명 포함

### ✅ 프리셋 활용 시나리오

- **브랜드 가이드**: 브랜드별 표준 프롬프트
- **캠페인 시리즈**: 시즌별/캠페인별 프리셋
- **개인 스타일**: 자주 쓰는 개인 스타일 저장

### ⚠️ 주의사항

- **보안**: 공유 링크는 읽기 전용이지만 URL이 노출됨
- **동기화**: 프리셋 수정 시 기존 링크는 변경되지 않음
- **의존성**: 공유된 블록이 삭제되면 프리셋에서도 사라짐
